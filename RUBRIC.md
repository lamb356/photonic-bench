# PhotonicBench System-Model And Pareto Rubric

Use this rubric for checklist completion decisions and the mandatory Hostile
Senior Reviewer critique.

## System Model Quality

- The model has explicit SRAM and off-chip/DRAM interface tiers.
- Tier assumptions are visible, documented, and schema-backed.
- Per-tier bytes, bandwidth-limited transfer time, and movement energy are
  computed deterministically.
- Compute energy and movement energy are separated in reports.
- Total system energy is presented as a local-model estimate, not a published
  result.
- Defaults are auditable and do not silently claim to represent any specific
  device.

## Visualizer And Pareto Quality

- Pareto charts answer practical trade-off questions:
  - energy/op vs throughput;
  - ops/byte vs latency;
  - system energy/movement share where available.
- Frontier points are computed deterministically and explained.
- Charts are readable on desktop and mobile.
- Missing legacy fields degrade clearly.
- The visualizer remains static-site friendly and generated by the CLI.
- The UI avoids decorative charting that does not improve analysis.

## Transformer Quality

- Full-transformer support handles multiple layers conveniently.
- Existing single-layer CLI/config behavior remains backward compatible.
- Generated full-model artifacts preserve layer/card provenance.
- Aggregate semantics are explicit for energy, timing, noise, memory, and
  exclusions.
- Tests cover representative multi-layer models and compatibility.

## Published Card Quality

- Each new card is backed by a published source with enough citation detail to
  find it again.
- Paper metrics stay in published-reference/provenance fields.
- Local workloads are explicitly named as surrogates.
- Missing source details are recorded as assumptions rather than guessed.
- Generated Markdown/JSON reports are committed with YAML configs.

## External Loading Quality

- Scope is intentionally limited and documented.
- Local/external loaded reports are validated before entering the UI.
- Invalid files fail clearly and do not corrupt generated artifact state.
- Existing generated visualizer behavior remains unchanged unless the user opts
  into loading external data.

## Documentation Quality

- README examples stay runnable.
- Model docs distinguish tier formulas, exclusions, and local assumptions.
- Schema docs stay aligned with generated JSON.
- Visualizer docs explain Pareto charts and frontier interpretation.
- Published-card docs preserve local-model versus paper-reference boundaries.

## Verification Quality

- Focused tests are run near the changed areas.
- Full Ruff and pytest are run before final completion.
- Visualizer generation is run after report/artifact changes.
- Browser smoke testing verifies the new visualizer workflow.
- Generated artifacts are inspected or asserted for stale/broken content.

## Review Readiness

- Checklist DONE items include proof.
- State files are current.
- The Hostile Senior Reviewer critique is recorded in `PROGRESS.md`.
- Important critique findings are fixed before completion or explicitly
  justified.
- Remaining risks are named plainly.

## Current Completion Assessment

- System model quality: met. Reports now include explicit SRAM and off-chip
  tiers, visible tier assumptions, deterministic movement energy/timing, and
  compute-versus-movement separation. Published paper numbers remain under
  `published_reference`.
- Visualizer and Pareto quality: met. Comparison mode exposes system metrics,
  exportable summaries, Pareto frontier detection, and automatic log scaling
  for wide positive axes.
- Transformer quality: met for the requested workflow convenience. Users can
  define counted multi-layer transformer models with representative layer
  artifacts and model summaries while preserving existing `transformer-layer`
  behavior.

  were added with DOI/source details, paper metrics, generated reports, and
  tests for provenance and surrogate boundaries.

  browser external loading validates the visualizer summary contract rather
